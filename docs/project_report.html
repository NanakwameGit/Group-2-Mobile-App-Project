1. Introduction

Managing personal finances is a challenge many students and young professionals face. The Personal Expense Tracker is a mobile application built with Flutter that allows users to log, categorize, and review their income and expenses in a clean, easy-to-use interface. All data is stored locally on the device — no internet connection, no account, and no server is required.

The app was designed with simplicity and presentation-readiness in mind: it uses a minimal number of files, no complex architecture, and relies only on Flutter's built-in widgets plus one small storage package (shared_preferences).

Goal: Build a working, good-looking Flutter app that demonstrates key mobile development concepts — state management, local persistence, navigation, and form validation — in the fewest possible lines of code.

Tap the delete icon on any row to remove it. The balance card updates instantly and the change is persisted.

Three-page intro shown only on first launch. Animated dot indicators, Skip link, and a "Get Started" button.

The project uses a deliberately flat structure to stay beginner-friendly:

lib/

    transaction.dart         — Transaction class + toJson / fromJson

4 Dart files total. No services layer, no repository pattern, no BLoC — just plain Flutter widgets with state.

5.2 State management

All mutable state (the transaction list) lives inside _HomeScreenState. Every user action calls setState(), which triggers a rebuild of the widget tree. There is no global state, no provider, and no stream — Flutter's built-in reactive model is sufficient for this scale.

5.3 Persistence

SharedPreferences stores a single key "transactions" whose value is a JSON array string. On every add or delete, the full list is re-encoded and written. On initState, the string is read back and decoded into a List<Transaction>. If the key is absent (first run), sample data is seeded and immediately saved.

7. UI Screens

7.1 Onboarding Screen

  • Three full-screen pages rendered by a PageView
  • Each page has a large icon in a tinted circle, a bold title, and a descriptive subtitle
  • Page colours: purple (track spending) → teal (categorize) → orange (big picture)
  • Animated dot indicators — active dot stretches to a pill shape
  • Next / Skip / Get Started controls; "Get Started" writes onboarding_done = true

7.2 Home Screen

  • Balance card: gradient container (primary → tertiary colour) with the balance in large type and income/expense tiles below
  • Transaction list: ListView.separated inside a SingleChildScrollView; each item is a Card with a ListTile
  • Empty state: centred message shown when the list is empty
  • FAB: FloatingActionButton.extended opens the add-transaction sheet

7.3 Add Transaction Sheet

  • Modal bottom sheet with rounded top corners
  • Padding grows with the keyboard via MediaQuery.viewInsets.bottom
  • Animated Expense / Income toggle buttons
  • Title and amount TextFormFields with inline validators
  • Category DropdownButtonFormField with 7 options
  • Date shown via InputDecorator; tapping opens showDatePicker
  • FilledButton submits and closes the sheet

9. How to Run

  1. Clone the repository: git clone https://github.com/NanakwameGit/Group-2-Mobile-App-Project.git
  2. Install dependencies: flutter pub get
  3. Run on a device or emulator: flutter run
  4. Run in Chrome: flutter run -d chrome
  5. Build for web: flutter build web
Requirements: Flutter SDK 3.38+ · Dart 3.10+ · An Android/iOS emulator or physical device (or Chrome for web)

10. Conclusion

The Personal Expense Tracker demonstrates that a fully functional, polished mobile app can be built with minimal complexity. By using StatefulWidget for state and shared_preferences for persistence, the codebase remains small enough to be explained in a single class session while still covering real-world mobile development concepts: navigation, form validation, async I/O, serialization, and first-launch onboarding.

The app is ready to be extended — potential next steps include charts/analytics, monthly budget limits, currency selection, or cloud sync — but it is deliberately kept minimal for the purposes of this presentation.
